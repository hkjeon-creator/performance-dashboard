# 퍼포먼스 마케팅 데이터 분석 가이드

## 프로젝트 개요

매일 채널 데이터(노출·클릭·비용)와 앱스플라이어 데이터(어트리뷰션 전환)를 수집해
조인·전처리 후 인사이트를 뽑는 퍼포먼스 마케팅 분석 워크플로우.

---

## 데이터 구조 & 파일 규칙

### 폴더 구조
```
data/
  channel/
    YYYY-MM-DD.parquet   # 채널 로우 데이터 (일별)
  appsflyer/
    YYYY-MM-DD.parquet   # 앱스플라이어 로우 데이터 (일별)
dashboard.py             # Streamlit 대시보드
```

### 파일 규칙
- 파일명: `YYYY-MM-DD.parquet` (날짜 형식 고정)
- 인코딩: UTF-8
- 날짜 컬럼명: `일`
- 신규 날짜 데이터 추가 시 두 폴더에 동시에 넣어야 조인 가능

### 채널 데이터 컬럼
| 컬럼 | 설명 |
|------|------|
| 일 | 날짜 |
| 채널 | 구글 / 메타 / 네이버 |
| 채널분류 | 외부 / 자체 |
| 캠페인 | 캠페인 코드명 |
| 캠페인목적 | 캠페인 목적 한글명 |
| 그룹 | 타겟 그룹 |
| 소재 | 소재 코드명 |
| 노출 / 클릭 / 비용 / 회원가입 / 구매 / 구매매출 | 수치 |

### 앱스플라이어 데이터 컬럼
| 컬럼 | 설명 |
|------|------|
| 일 | 날짜 |
| 미디어소스 | googleadwords_int / Facebook Ads / naver_search |
| 캠페인 / 그룹 / 소재 | 채널 데이터와 동일한 값 |
| 클릭 / 회원가입 / 구매 / 구매매출 | 어트리뷰션 기준 수치 |

## 채널 - 미디어소스 매핑

두 데이터를 조인할 때 반드시 아래 매핑 적용.

| 채널 (channel) | 미디어소스 (appsflyer) |
|----------------|----------------------|
| 구글 | googleadwords_int |
| 메타 | Facebook Ads |
| 네이버 | naver_search |

조인 키: `날짜 + 미디어소스(매핑 후) + 캠페인 + 그룹 + 소재`

## 네이밍 컨벤션

### 캠페인 코드
```
{채널코드}_CMP_{번호}_{목적}
예: GGL_CMP_01_플러스가입
```

| 채널코드 | 채널 |
|----------|------|
| GGL | 구글 |
| META | 메타 |
| NVR | 네이버 |

### 캠페인 목적 분류
| 캠페인 | 목적 | 퍼널 단계 |
|--------|------|-----------|
| GGL_CMP_01_플러스가입 / META_CMP_01_신규유저 | 신규 회원 획득 | 상단 |
| GGL_CMP_03_첫구매 | 첫 구매 유도 | 중단 |
| GGL_CMP_02_리타겟팅 | 재방문 유도 | 중단 |
| META_CMP_02_룩얼라이크 | 유사 타겟 확장 | 상단 |
| META_CMP_03_재구매 / NVR_CMP_03_재구매 | 재구매·리텐션 | 하단 |
| NVR_CMP_01_브랜드KW | 브랜드 검색 방어 | 하단 |
| NVR_CMP_02_일반KW | 비브랜드 검색 획득 | 상단~중단 |

> **주의**: 퍼널 단계가 다른 캠페인을 ROAS 단일 지표로 비교하지 말 것.
> 신규획득 캠페인은 CPA·CVR, 리텐션 캠페인은 ROAS·매출 기준으로 평가.

### 그룹(타겟) 분류
| 그룹명 | 의미 |
|--------|------|
| 논타겟 | 타겟팅 없는 broad 오디언스 |
| 유사타겟 | lookalike 오디언스 |
| 리마케팅 | 사이트/앱 방문자 리타겟 |
| VIP | 고가치 기존 고객 |
| 윈백 | 이탈 고객 재활성화 |

### 소재 코드
```
{소재타입}_{카테고리}_{시즌}_{AB버전}_{버전번호}
예: VID_플러스멤버십_겨울_A_v1
```

| 소재타입 | 형식 |
|----------|------|
| VID | 영상 |
| IMG | 이미지 |
| CRS | 캐러셀 (카드뉴스) |
| TXT | 텍스트 (검색 광고) |

| 파트 | 규칙 |
|------|------|
| 카테고리 | 플러스멤버십 / 적립혜택 / 할인쿠폰 / 배송혜택 / 특가 / 신상품 |
| 시즌 | 겨울 / 봄 / 여름 / 가을 또는 날짜코드 |
| AB버전 | `_A_` / `_B_` 있으면 AB테스트 소재. 없으면 단독 소재 |
| 버전번호 | v1, v2 … 동일 컨셉의 수정 버전 |

> AB테스트 소재 분석 시 반드시 A/B를 쌍으로 비교할 것.

## KPI 정의 & 계산식

| KPI | 계산식 | 단위 | 기준 소스 |
|-----|--------|------|-----------|
| CTR | 클릭 / 노출 × 100 | % | channel |
| CVR | 구매 / 클릭 × 100 | % | channel |
| CPC | 비용 / 클릭 | 원 | channel |
| CPA | 비용 / 구매 | 원 | channel |
| ROAS | 구매매출 / 비용 × 100 | % | channel |
| 회원가입률 | 회원가입 / 클릭 × 100 | % | channel |

> ROAS는 **%** 단위 사용 (300% = 3배). 배수 표기 금지.

### 데이터 신뢰 소스 우선순위
| 지표 | 신뢰 소스 | 이유 |
|------|-----------|------|
| 노출 / 비용 | channel | appsflyer에 없음 |
| 구매 / 매출 | channel | 매체 보고 기준 |
| 클릭 | appsflyer | 어트리뷰션 클릭 기준 |
| 회원가입 | channel | 매체 보고 기준 |

두 소스 간 구매 수치 차이 발생 시: channel 기준으로 보고, appsflyer는 참고값으로 병기.

### 이상치 판단 기준
| 지표 | 이상 범위 | 조치 |
|------|-----------|------|
| CTR | < 0.1% | 소재·타겟팅 문제 의심 |
| CVR | > 20% | 데이터 오류 의심, 원본 확인 |
| ROAS | < 100% | 적자 캠페인, 즉시 보고 |
| ROAS | > 5000% | 데이터 이상 또는 소량 집행 의심 |

## 분석 원칙

1. **퍼널별 KPI 분리**: 신규획득(CPA, 회원가입률) vs 리텐션(ROAS, 매출) 혼용 금지
2. **AB테스트 소재**: A/B 쌍으로만 비교. 단독 비교 금지
3. **소재 유형별 기준 분리**: TXT(검색)는 CTR 기준 높음, VID는 낮음 — 유형 간 CTR 직접 비교 금지
4. **채널분류 구분**: 외부(구글·메타) vs 자체(네이버) — 비용 효율 비교 시 분류 명시
5. **수치 표기**: 비용·매출은 원 단위 천 단위 콤마, 비율은 소수점 둘째 자리

## 자주 쓰는 분석 패턴

```python
# 채널-앱스플라이어 조인
MEDIA_MAP = {"구글": "googleadwords_int", "메타": "Facebook Ads", "네이버": "naver_search"}
ch["미디어소스_af"] = ch["채널"].map(MEDIA_MAP)
merged = ch.merge(af, on=["날짜", "미디어소스_af", "캠페인", "그룹", "소재"], how="left")

# 소재 타입 파싱
df["소재타입"] = df["소재"].str.split("_").str[0]  # VID / IMG / CRS / TXT

# AB테스트 여부
df["is_AB"] = df["소재"].str.contains(r"_[AB]_", regex=True)

# 시즌 파싱
df["시즌"] = df["소재"].str.extract(r"_(겨울|봄|여름|가을)_")
```
